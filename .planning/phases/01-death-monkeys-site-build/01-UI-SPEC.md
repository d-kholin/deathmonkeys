## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline pill spacing |
| sm | 8px | Compact element spacing, card inner padding top/bottom |
| md | 16px | Default element spacing, card padding, nav link gaps |
| lg | 24px | Section padding, card grid gap |
| xl | 32px | Layout horizontal padding, hero content gap |
| 2xl | 48px | Major section breaks, hero padding |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Mobile hamburger tap target: minimum 44px × 44px (touch accessibility minimum)
- Nav bar height: 56px (fits wordmark + links at 16px with 20px vertical padding each side)

> Source: 8-point scale default; hamburger exception from CONTEXT.md D-06 (hamburger toggle); nav height at designer discretion per CONTEXT.md

|------|------|--------|-------------|------|

| Label | 14px | 400 (regular) | 1.4 | ui-sans-serif, system-ui, sans-serif |

Additional rules:

- Real name in RosterCard: Label scale (14px, 400) — secondary, muted
- Status badge text: Label scale (14px, 400), uppercase, letter-spacing 0.05em
- Hero tagline "Armed. Dangerous. Slightly Bananas.": Display scale (40px) on desktop; 28px on mobile (< 640px)

## Interaction States

### Mobile Navigation (CONTEXT.md D-06, D-07)
- Hamburger button: 44×44px tap target, positioned top-right in nav
- Open: panel slides down below nav bar, links stack vertically at 16px padding each, full viewport width
- Close trigger: tapping any nav link auto-closes (vanilla JS `<script is:inline>`)
- Deferred: click-outside-to-close, open/close CSS transition animation (per CONTEXT.md deferred list)

### Cards (hover)
- EventCard: no transform; left border changes from transparent to `#6b7c4a` on hover (2px solid)
- RosterCard: no transform; opacity of inactive entries is 0.5, no hover change for inactive
- GalleryCard: border color transitions to `#6b7c4a` on hover (2px solid, replaces `#1f2937` default)

### Links
- Nav links: default color `#d1d5db` (gray-300), hover color `#ffffff`, active page underline `#6b7c4a`
- Body links: not used in v1 (content is markdown prose — standard underline, inherit color)

## Tailwind v4 Token Declarations (`@theme`)

The following custom properties must be declared in the `@theme` block of the global CSS file. Tailwind v4 generates utility classes automatically from these.

```css
@theme {
  /* Colors */
  --color-surface: #0a0a0a;
  --color-surface-raised: #141414;
  --color-surface-inactive: #1f1f1f;
  --color-accent: #6b7c4a;
  --color-destructive: #7f2e2e;
  --color-badge-completed: #3f3f46;
  --color-text-primary: #f4f4f5;
  --color-text-secondary: #a1a1aa;
  --color-text-muted: #6b7280;
  --color-border: #27272a;

  /* Typography */
  --font-family-heading: 'JetBrains Mono', monospace;
  --font-family-body: ui-sans-serif, system-ui, sans-serif;

  /* Spacing (8-point scale) */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;
  --spacing-2xl: 48px;
  --spacing-3xl: 64px;

  /* Nav */
  --nav-height: 56px;
}
```

Google Fonts import (top of global CSS, before `@import "tailwindcss"`):
```css
@import url('https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;700&display=swap');
```

> Source: REQUIREMENTS.md FOUND-02; CONTEXT.md D-04; CLAUDE.md Tailwind v4 `@theme` directive
