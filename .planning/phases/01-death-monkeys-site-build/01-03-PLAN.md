<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Inline `<script is:inline>` → browser DOM | Client-side JS manipulates nav state at runtime |
| Layout title prop → rendered `<head>` | Page-supplied title string injected into `<title>` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | XSS via `title` prop in `<head>` | mitigate | Astro escapes `{title}` interpolation by default; title values are first-party literals from page frontmatter, never user input. No `set:html`. |
| T-03-02 | Tampering | Inline nav script | accept | `<script is:inline>` toggles only local DOM class/aria state; no eval, no external fetch, no DOM injection of untrusted strings. Static site, no user input. |
| T-03-03 | Denial of Service | Hamburger script error blocks nav | mitigate | Toggle logic guards on element existence; desktop links remain plain anchors (work without JS), so navigation degrades gracefully if the script fails. |
</threat_model>
